Voce vai desenvolver um sistema web para meu trabalho de extensao.

Projeto:
Gerenciador de financas pessoal.

Repositorio obrigatorio:
https://github.com/Aliceria/financemanager

Branch obrigatoria:
main

Objetivo do sistema:
Criar um gerenciador de financas pessoal completo, simples de usar, com foco em organizacao financeira, previsao de gastos e acompanhamento mensal.

Regra principal de privacidade:
O sistema NAO deve vender, compartilhar, analisar externamente ou tratar os dados financeiros do usuario para nenhum fim alem do funcionamento local do proprio sistema.

Os dados do usuario devem ser armazenados de forma segura, usando criptografia sempre que possivel. Nenhum dado sensivel deve ficar exposto em texto puro sem necessidade.

Funcionalidades principais:

1. Tela inicial

* Ao entrar no aplicativo, o sistema deve abrir automaticamente na data atual.
* O usuario deve conseguir selecionar qualquer data ate 1 ano para frente.
* Ao selecionar uma data futura, o sistema deve mostrar uma expectativa/previsao das financas do usuario ate aquele periodo.

2. Renda

* O usuario deve conseguir definir sua renda mensal total.
* Essa renda sera usada como base para calculos de gastos fixos, variaveis e percentuais.

3. Tipos de gastos
   O sistema deve permitir cadastrar 3 tipos principais de gasto:

a) gasto fixo em dinheiro

* Exemplo: aluguel de 800 reais, internet de 100 reais, assinatura de 20 reais.
* O usuario informa um valor fixo em dinheiro.
* Esse valor deve ser descontado mensalmente da renda.

b) gasto percentual da renda

* Exemplo: 10% da renda em investimento, 15% em lazer, 5% em reserva.
* O usuario informa uma porcentagem da renda.
* O sistema deve calcular automaticamente quanto esse percentual representa em dinheiro.
* O sistema deve mostrar quanto esta sendo gasto e quanto sobra da renda.

c) gasto variavel mensal

* Exemplo: mercado, gasolina, lazer, farmacia.
* O usuario define o valor do gasto variavel em um mes.
* Esse valor deve continuar sendo considerado nos meses seguintes usando a ultima definicao feita.
* O valor so muda quando o usuario atualizar manualmente.

4. Previsao financeira

* O sistema deve calcular a expectativa financeira para os proximos meses, ate no maximo 1 ano para frente.
* Deve mostrar:

  * renda total
  * total de gastos fixos
  * total de gastos percentuais
  * total de gastos variaveis
  * saldo previsto
  * diferenca entre meses, quando aplicavel

5. Interface

* A interface deve ser simples, limpa e facil de entender.
* Deve ser adequada para apresentacao academica.
* O usuario deve conseguir adicionar, editar e remover gastos.
* O usuario deve conseguir alterar renda.
* O usuario deve conseguir navegar por datas/meses.

6. Restricoes tecnicas

* Nao usar servicos pagos.
* Nao depender de APIs externas para funcionalidades essenciais.
* Nao criar funcionalidades desnecessariamente complexas.
* Priorizar codigo simples, organizado e facil de explicar.
* O sistema deve funcionar localmente ou em ambiente web simples.
* Caso use banco de dados, os dados sensiveis devem ser protegidos com criptografia ou tecnica equivalente adequada ao projeto.
* Nunca expor chaves, tokens, senhas ou dados sensiveis no codigo.
* Nunca commitar arquivos .env, node_modules, dist, build ou arquivos temporarios.

7. Git e GitHub
   Trabalhe apenas neste repositorio GitHub:

Repositorio: https://github.com/Aliceria/financemanager
Branch padrao: main

Antes de fazer qualquer alteracao:

1. Confirme que o diretorio atual e um repositorio Git.
2. Confirme que o resultado de `git remote -v` aponta para https://github.com/Aliceria/financemanager.
3. Se nao apontar para esse repositorio, pare imediatamente e me informe.
4. Nunca inicialize um novo repositorio Git sem minha autorizacao.

Depois de cada feature implementada:

1. Execute `git status`.
2. Execute testes ou verificacoes relevantes, se existirem.
3. Faca commit apenas dos arquivos relacionados a feature implementada.
4. Use uma mensagem de commit clara.
5. Faca push apenas para a branch `main`.
6. Nao faca push para nenhum outro repositorio, remote ou branch.

Regra de commits:

* Fazer commits pequenos e organizados.
* Fazer um commit ao finalizar cada feature funcional.
* Nao juntar muitas features diferentes no mesmo commit.
* Antes de cada commit, mostrar um pequeno resumo do que foi alterado.

Mensagens de commit:

* Devem estar obrigatoriamente em portugues.
* Devem conter apenas letras minusculas.
* Nao devem conter acentos, cedilhas ou caracteres especiais.
* Devem ser curtas, claras e descritivas.

Exemplos de commits validos:

* criar estrutura inicial do projeto
* adicionar tela inicial
* implementar cadastro de renda
* adicionar cadastro de gastos fixos
* implementar gastos percentuais
* criar gastos variaveis mensais
* adicionar previsao financeira
* ajustar layout principal
* proteger dados do usuario
* atualizar documentacao

Operacoes proibidas sem minha autorizacao explicita:

* git push --force
* git push --force-with-lease
* git reset --hard
* git rebase interativo
* apagar historico de commits
* deletar branch
* alterar remote
* commitar arquivos sensiveis
* commitar tokens, senhas ou chaves de API

Fluxo esperado:

1. Verificar repositorio e branch.
2. Planejar a proxima feature.
3. Implementar a feature.
4. Testar/verificar.
5. Mostrar resumo.
6. Commitar com mensagem valida.
7. Fazer push para main.
8. Continuar para a proxima feature.

Comece implementando a estrutura inicial do projeto e depois siga feature por feature.

8. Sincronizacao inicial obrigatoria

Ao iniciar o trabalho, antes de implementar qualquer nova funcionalidade:

1. Analise todo o conteudo atual da pasta do projeto.
2. Identifique arquivos criados ou modificados que ainda nao estejam commitados.
3. Execute `git status` para verificar o estado atual do repositorio.
4. Caso existam alteracoes nao commitadas:
   - Agrupe apenas os arquivos relacionados ao estado atual do projeto.
   - Crie um commit inicial representando exatamente o estado atual encontrado.
   - Faca push desse commit para a branch `main`.
5. Somente apos sincronizar completamente o estado atual do projeto voce pode comecar a implementar novas funcionalidades.

Mensagem de commit sugerida para essa sincronizacao inicial:
- sincronizar estado atual do projeto

Importante:
- Nao descartar alteracoes existentes.
- Nao sobrescrever trabalho ja realizado.
- Nao recriar arquivos que ja existem.
- Tratar o conteudo atual da pasta como a fonte de verdade inicial do projeto.
- Sempre preservar todo o trabalho existente antes de iniciar novas implementacoes.
